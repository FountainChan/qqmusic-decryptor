## 文档信息
- **创建日期**: 2026-01-31
- **适用范围**: 整个项目

## 一、备份管理原则

### 1.1 核心原则

1. **优先使用 Git**: 所有代码修改都通过 Git 进行版本管理
2. **临时备份归档**: 所有临时备份文件（.bak, .backup）移到 `drops/backup/` 目录
3. **定期清理**: 定期检查 `drops/backup/` 目录，删除过期的备份
4. **规则约束**: 遵循 .gitignore 规则，不提交临时文件

### 1.2 备份类型

| 备份类型 | 存放位置 | 保留期限 | 管理方式 |
|---------|---------|---------|---------|
| Git 提交 | Git 仓库 | 永久 | Git 管理 |
| 临时备份 | `drops/backup/` | 30天 | 手动清理 |
| 重要备份 | `drops/backup/` | 长期 | 手动清理 |

## 二、备份文件管理

### 2.1 临时备份规则

#### 文件位置
```
drops/backup/
├── config.ini.backup          # 配置文件备份
├── main_cli.py.backup        # CLI程序备份
├── main_gui.py.backup        # GUI程序备份
└── main_gui.py.bak          # GUI程序临时备份
```

#### 保留期限
- **临时备份**: 保留 30 天
- **重要备份**: 保留 90 天
- **过期处理**: 超过保留期限的备份应删除

#### 备份命名规则

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 配置文件 | `config.ini.backup` | config.ini.backup |
| Python 文件 | `filename.py.backup` | main_cli.py.backup |
| 临时文件 | `filename.py.bak` | main_gui.py.bak |

### 2.2 备份移动规则

#### 新备份创建流程

1. **创建备份前**: 检查 `drops/backup/` 目录空间
2. **创建备份**: 使用 `.backup` 后缀创建备份
3. **移动备份**: 立即移动到 `drops/backup/` 目录
4. **记录日志**: 记录备份创建的时间和原因

#### 示例流程

```bash
# 1. 创建备份
cp main_cli.py main_cli.py.backup

# 2. 移动备份
mv main_cli.py.backup drops/backup/

# 3. Git 提交（如果需要）
git add main_cli.py
git commit -m "修复统计逻辑"
```

### 2.3 备份清理规则

#### 清理频率
- **手动清理**: 每月检查一次
- **自动清理**: 可以创建清理脚本定期执行

#### 清理标准

| 保留期限 | 删除条件 | 示例 |
|---------|---------|------|
| 30天 | 超过30天且无特殊标记 | 临时备份 |
| 90天 | 超过90天 | 重要备份 |
| 永久 | 重要的历史版本 | 不删除 |

#### 清理命令示例

```bash
# 查找超过30天的备份文件
find drops/backup/ -name "*.backup" -mtime +30 -type f

# 删除超过30天的备份
find drops/backup/ -name "*.backup" -mtime +30 -type f -delete

# 查找超过90天的备份
find drops/backup/ -name "*.backup" -mtime +90 -type f
```

## 三、Git 备份管理

### 3.1 Git 提交原则

#### 何时使用 Git

| 场景 | Git 提交 | 临时备份 |
|------|---------|---------|
| 功能开发 | ✅ 使用 | ❌ 不需要 |
| Bug 修复 | ✅ 使用 | ❌ 不需要 |
| 重构代码 | ✅ 使用 | ❌ 不需要 |
| 实验性修改 | ⚠️ 谨慎使用 | ✅ 建议使用 |
| 小调整 | ❌ 不必要 | ✅ 建议使用 |

#### Git 提交规范

```bash
# 功能开发
git add .
git commit -m "feat: 添加统计功能"

# Bug 修复
git add .
git commit -m "fix: 修复统计数字错误"

# 重构
git add .
git commit -m "refactor: 优化代码结构"
```

### 3.2 .gitignore 规则

#### 当前规则

```
nul                         # Windows 空设备
gui_backup                  # GUI 备份目录
__pycache__/               # Python 缓存
*.bak                       # 临时备份文件
*.backup                    # 备份文件
drops/                      # drops 目录（备份等）
logs/                       # 日志目录
*.pyc                       # Python 字节码
*.pyo                       # Python 优化字节码
*.log                       # 日志文件
*.exe.xz                    # 压缩的可执行文件
```

#### 规则说明

| 规则 | 说明 | 原因 |
|------|------|------|
| `*.bak` | 忽略所有 .bak 文件 | 临时备份不应提交 |
| `*.backup` | 忽略所有 .backup 文件 | 临时备份不应提交 |
| `drops/` | 忽略整个目录 | 临时文件和备份 |
| `logs/` | 忽略日志目录 | 日志不应提交 |
| `__pycache__/` | 忽略 Python 缓存 | 自动生成的文件 |

## 五、自动化脚本建议

### 5.1 备份清理脚本

创建 `drops/cleanup_backups.sh`:

```bash
#!/bin/bash
# 备份清理脚本

BACKUP_DIR="drops/backup"
RETENTION_DAYS=30

echo "查找超过 $RETENTION_DAYS 天的备份文件..."
find $BACKUP_DIR -name "*.backup" -mtime +$RETENTION_DAYS -type f -print

read -p "是否删除这些文件？(y/n): " confirm
if [ "$confirm" = "y" ]; then
    find $BACKUP_DIR -name "*.backup" -mtime +$RETENTION_DAYS -type f -delete
    echo "清理完成"
else
    echo "清理已取消"
fi
```

### 5.2 备份管理脚本

创建 `drops/manage_backups.sh`:

```bash
#!/bin/bash
# 备份管理脚本

BACKUP_DIR="drops/backup"

echo "===== 备份文件清单 ====="
ls -lh $BACKUP_DIR/
echo ""

echo "===== 磁盘占用 ====="
du -sh $BACKUP_DIR/
echo ""

echo "===== 超过 30 天的备份 ====="
find $BACKUP_DIR -name "*.backup" -mtime +30 -type f -exec ls -lh {} \;
```

## 六、备份管理最佳实践

### 6.1 日常操作

#### 开发流程

1. **修改代码**
   - 使用 Git 提交
   - 或创建临时备份（.bak）

2. **移动备份**
   - 立即移动到 `drops/backup/` 目录

3. **定期清理**
   - 每月检查一次
   - 删除过期备份

#### 错误处理

1. **发现错误**
   - 先 Git 回退
   - 或从 `drops/backup/` 恢复

2. **恢复后**
   - 删除不需要的备份
   - 重新创建正确版本

### 6.2 备份记录

#### 记录方式

| 记录类型 | 位置 | 说明 |
|---------|------|------|
| Git 提交 | Git 仓库 | 自动记录 |
| 临时备份 | `drops/backup/` | 文件名包含日期 |
| 重要备份 | `drops/backup/` | 文件名包含日期和说明 |

#### 记录建议

- 备份文件名包含日期（可选）
- Git 提交信息清晰明确
- 重要备份添加说明文件

### 6.3 安全性考虑

#### 备份安全

| 安全措施 | 说明 |
|---------|------|
| 备份位置 | `drops/backup/` 目录不在 Git 仓库 |
| 敏感信息 | 不备份包含密钥的配置文件 |
| 访问权限 | 确保备份文件的适当权限 |

## 七、常见问题

### 7.1 备份相关

**Q1: 何时使用 Git，何时使用临时备份？**

A: 
- **Git 提交**: 用于功能开发、Bug 修复、重构等正式修改
- **临时备份**: 用于实验性修改、小调整、不确定的修改

**Q2: 如何删除过期备份？**

A: 使用提供的清理脚本或手动执行：
```bash
find drops/backup/ -name "*.backup" -mtime +30 -type f -delete
```

**Q3: 如何恢复备份？**

A: 从 `drops/backup/` 目录复制文件：
```bash
cp drops/backup/main_cli.py.backup main_cli.py
```

### 7.2 Git 相关

**Q4: 如何查看 Git 历史记录？**

A: 使用 Git 日志命令：
```bash
git log --oneline
git log --graph
```

**Q5: 如何回退到特定版本？**

A: 使用 Git 回退命令：
```bash
git checkout <commit-hash>
git reset --hard <commit-hash>
```
