### 关键问题

**核心问题**：当找不到 `albummid` 时，所有文件都被跳过了（`跳过: 575`），而不是专辑被跳过（`跳过: 0`）

**根本原因**：缺少 `return stats` 语句，导致处理继续

**修改后**：
```python
    # 查询 API 获取专辑信息
    try:
        if not api_cache:
            album_info = search_album_by_name(f"{artist} {album}")
        else:
            # 检查缓存
            cache_key = f"{artist} - {album}"
            if cache_key in api_cache:
                album_info = api_cache[cache_key]
                logger.info(f"专辑信息已缓存")

        logger.error(f"查询专辑信息异常: {e}")

    # 获取发行年份

    cover_data = None
```

**关键改进**：
- ✅ 添加 `try-except` 块
- ✅ 捕获所有 API 调用异常
- ✅ 设置 `album_info = None`，触发后续跳过逻辑
- ✅ 添加详细的错误日志
