### 核心问题

**问题**：当找不到 `albummid` 时，所有文件都被跳过，**音轨号也无法写入**

**影响**：
- ❌ 575 个文件的音轨号全部丢失
- ❌ 音轨号处理完全失效
- ❌ 用户无法看到任何处理进度

---

## 🔍 根本原因分析

### 问题 1：提前返回导致文件处理循环无法执行

**代码位置**：`supplement_album_metadata.py` 第 618-621 行

```python
albummid = search_result.get("albummid")
if not albummid:
    logger.warning(f"未找到 albummid")
    stats["skipped"] = len(audio_files)  # 设置跳过数为 575
    return stats  # ← 直接返回，退出函数！
```

**执行流程**：

```
1. 查询 API 获取专辑信息...
2. 未找到 albummid
3. 设置 stats["skipped"] = 575
4. return stats  ← 函数返回
5. 【文件处理循环永远不会执行】
   - 第 661 行：for i, audio_file in enumerate(audio_files):
   - 所有文件的音轨号都无法写入
```

**根本原因**：
- ❌ 当找不到 `albummid` 时，代码直接 `return stats`
- ❌ 第 661 行开始的文件处理循环永远不会执行
- ❌ 所有文件都被跳过，包括音轨号处理

---

### 问题 2：代码结构混乱

**当前代码有两处 API 调用**：

| 位置 | 代码 | 功能 |
|------|------|------|
| **第 607-647 行**（循环外） | `search_album()` + `get_album_info()` | 用于缓存 |
| **第 680-731 行**（循环内） | `search_album()` + `get_album_info()` | 每个文件都调用 |

**问题**：
- ❌ 第 680 行的 API 调用在循环内，重复调用 575 次
- ❌ 当第 621 行提前返回时，第 680 行的代码永远不会执行
- ❌ 代码逻辑混乱，难以维护

---

### 问题 3：违背优化方案的核心要求

| 优化方案要求 | 当前状态 | 问题 |
|-------------|----------|------|
| **音轨号总是写入**（不依赖 API） | ❌ 找不到 albummid 时直接返回，不写入 | **严重违背** |
| **年份和封面依赖 API**（失败则跳过） | ✅ 依赖 API | 符合 |
| **音轨号和年份封面独立处理** | ❌ API 调用失败时，音轨号也无法处理 | **严重违背** |

**关键问题**：
- ❌ **音轨号处理和 API 调用混在一起**
- ❌ **API 调用失败导致音轨号也无法写入**
- ❌ **违背了"音轨号总是写入"的核心要求**

---

## 🔧 修复方案

### 方案概述

**核心思想**：重构 `process_album_directory()` 函数，确保音轨号处理独立于 API 调用

**修改原则**：
1. ✅ 移除提前返回逻辑
2. ✅ 音轨号处理不依赖 API
3. ✅ 年份和封面处理依赖 API，失败则跳过
4. ✅ 删除重复的 API 调用

---

### 详细实施步骤

#### 步骤 1：重构主处理函数

**修改文件**：`supplement_album_metadata.py`

**修改位置**：`process_album_directory()` 函数（第 546-740 行）

**修改逻辑**：

**修改前**（第 617-621 行）：
```python
albummid = search_result.get("albummid")
if not albummid:
    logger.warning(f"未找到 albummid")
    stats["skipped"] = len(audio_files)  # ← 跳过所有文件
    return stats  # ← 直接返回，不处理文件
```

**修改后**：
```python
albummid = search_result.get("albummid")
if not albummid:
    logger.warning("未找到 albummid")
    logger.warning("年份和封面处理将被跳过，但音轨号仍然会处理")
    pub_year = None
    cover_data = None
    # 不返回，继续处理文件（音轨号仍然会写入）
```

#### 步骤 2：分离音轨号和年份封面的处理

**修改位置**：第 660-731 行

**修改前**：
```python
# 处理所有音频文件（FLAC 和 OGG）
for i, audio_file in enumerate(audio_files):
    filename = os.path.basename(audio_file)
    logger.info(f"[INFO] 处理文件 {i+1}/{len(audio_files)}: {filename}")
    
    # 步骤 1：音轨号处理（不依赖 API，总是尝试）
    track_number = extract_track_number_from_filename(filename)
    if track_number is not None:
        success = write_track_number(audio_file, track_number)
        if success:
            stats["success"] += 1
            logger.info(f"[OK] 音轨号: {track_number}")
        else:
            stats["failed"] += 1
            logger.warning(f"[FAIL] 音轨号: {track_number}")
    else:
        logger.warning("[SKIP] 未找到音轨号")
        stats["skipped"] += 1
    
    # 步骤 2：年份和封面处理（依赖于 API）
    try:
        # 提取标签
        tags = extract_audio_tags(audio_file)
        artist = tags.get("artist")
        album = tags.get("album")
        
        # 调用 API 获取专辑信息
        search_result = search_album(artist, album)
        
        if search_result and "albummid" in search_result:
            # 获取年份和封面
            albummid = search_result["albummid"]
            album_info = get_album_info_with_date(albummid)
            
            if album_info:
                # 提取年份
                pub_year = album_info.get("pub_year")
                if pub_year:
                    logger.info(f"找到年份: {pub_year}")
                else:
                    logger.warning("未找到发行年份")
                
                # 提取封面
                cover_data = download_cover(albummid)
                if cover_data:
                    logger.info(f"封面数据大小: {len(cover_data)} bytes")
                else:
                    logger.warning("下载封面失败")
                
                # 嵌入元数据（年份 + 封面）
                if pub_year or cover_data:
                    success = embed_year_and_cover(audio_file, pub_year, cover_data)
                    if success:
                        stats["success"] += 1
                        logger.info(f"[OK] {filename}")
                    else:
                        stats["failed"] += 1
                        logger.warning(f"[FAIL] {filename}")
                else:
                    logger.warning("专辑详情为空")
                    stats["failed"] += 1
            else:
                logger.warning("未找到专辑详情")
                stats["failed"] += 1
        else:
            logger.warning("未找到 albummid")
            logger.warning("跳过专辑：未找到发行年份")
            # 不返回，继续下一个文件
    except Exception as e:
        logger.error(f"处理异常 {filename}: {e}")
        stats["failed"] += 1
```

**修改后**：
```python
# 处理所有音频文件（FLAC 和 OGG）
for i, audio_file in enumerate(audio_files):
    filename = os.path.basename(audio_file)
    logger.info(f"[INFO] 处理文件 {i+1}/{len(audio_files)}: {filename}")
    
    # 步骤 1：音轨号处理（不依赖 API，总是尝试）
    track_number = extract_track_number_from_filename(filename)
    if track_number is not None:
        success = write_track_number(audio_file, track_number)
        if success:
            stats["success"] += 1
            logger.info(f"[OK] 音轨号: {track_number}")
        else:
            stats["failed"] += 1
            logger.warning(f"[FAIL] 音轨号: {track_number}")
    else:
        logger.warning("[SKIP] 未找到音轨号")
        stats["skipped"] += 1
    
    # 步骤 2：年份和封面处理（依赖 API，可能为 None）
    if pub_year or cover_data:
        try:
            success = embed_year_and_cover(audio_file, pub_year, cover_data)
            if success:
                stats["success"] += 1
                logger.info(f"[OK] {filename}")
            else:
                stats["failed"] += 1
                logger.warning(f"[FAIL] {filename}")
        except Exception as e:
            logger.error(f"嵌入年份和封面失败 {filename}: {e}")
            stats["failed"] += 1
    else:
        logger.info("[SKIP] 没有年份和封面数据")
```

#### 步骤 3：删除重复的 API 调用

**修改位置**：第 607-647 行 和 第 680-731 行

**修改逻辑**：

**修改前**：
- 在循环外调用一次 API（第 607-647 行）
- 在循环内又调用一次 API（第 680-731 行）

**修改后**：
- 只在循环外调用一次 API
- 在循环内直接使用缓存的数据（`pub_year` 和 `cover_data`）

## 🔧 修复方案详情

### 关键问题

**问题 1**：统计数字不准确
- 总文件数显示为 7（实际是 14）
- 成功数为 0（实际是 13）

**问题 2**：统计逻辑重复累加
- 在处理文件循环中重复累加 `stats["total"]`
- 导致统计数字异常

**根本原因**：
```python
# 修改前：在循环内重复累加统计
for i, audio_file in enumerate(audio_files):
    stats["total"] += 1  # ❌ 重复累加
    # ... 处理文件
```

### 修复方案

**核心修改**：将统计逻辑从循环内移出，预先设置总文件数

```python
# 修改后：在循环外预先设置总文件数
total_files = sum(len(files) for files in dir_audio_files.values())
stats["total"] = total_files  # ✅ 只设置一次

for dir_path, audio_files in sorted(dir_audio_files.items()):
    # 处理每个子目录
    for i, audio_file in enumerate(audio_files):
        # 处理文件
        # 不重复累加 stats["total"]
```
