## 📊 优化方案详情

### 方案 1：拆分元数据处理

**核心思想**：将元数据处理分为三个独立的部分

| 部分 | 功能 | 依赖 API | 失败处理 |
|------|------|----------|----------|
| **音轨号处理** | 从文件名提取音轨号，直接写入 | ❌ 不依赖 | ✅ 总是尝试 |
| **年份和封面处理** | 从 API 获取年份和封面，写入 | ✅ 依赖 API | ✅ 失败则跳过 |

**处理逻辑**：
```python
# 步骤 1：音轨号处理（总是尝试）
track_number = extract_track_number_from_filename(filename)
if track_number is not None:
    # 写入音轨号（不依赖 API）
    success = write_track_number(audio_file, track_number)
    if success:
        logger.info(f"[OK] 音轨号: {track_number}")
        stats["success"] += 1
    else:
        logger.warning(f"[FAIL] 音轨号: {track_number}")
        stats["failed"] += 1
    continue  # 成功写入音轨号后，继续下一个文件
else:
    logger.warning(f"[SKIP] 未找到音轨号")
    stats["skipped"] += 1
    continue  # 未找到音轨号，跳过

# 步骤 2：年份和封面处理（依赖于 API）
try:
    # 获取专辑信息
    tags = extract_audio_tags(audio_file)
    artist = tags.get("artist")
    album = tags.get("album")
    
    # 调用 API 获取专辑信息
    album_info = search_album(artist, album)
    
    if album_info and "albummid" in album_info:
        # 获取年份和封面
        albummid = album_info["albummid"]
        album_details = get_album_info(albummid)
        
        if album_details:
            # 提取年份
            pub_year = album_details.get("pub_year")
            
            # 获取封面
            cover_data = download_cover(albummid)
            
            # 嵌入元数据（年份 + 封面）
            if pub_year or cover_data:
                success = embed_year_and_cover(audio_file, pub_year, cover_data)
                if success:
                    logger.info(f"[OK] {filename}")
                    stats["success"] += 1
                else:
                    logger.warning(f"[FAIL] {filename}")
                    stats["failed"] += 1
        else:
            logger.warning("专辑详情为空")
            stats["failed"] += 1
    else:
        logger.warning("未找到 albummid")
        logger.warning("跳过专辑：未找到发行年份")
        # 不返回，继续下一个文件
except Exception as e:
    logger.error(f"处理异常 {filename}: {e}")
    stats["failed"] += 1
```

**关键改进**：
- ✅ 音轨号处理独立于 API 调用
- ✅ 年份和封面处理依赖 API 调用
- ✅ 添加了异常处理（try-except 块）
- ✅ 添加了跳过逻辑（当找不到 `albummid` 时）
- ✅ 添加了详细的日志输出

### 方案 2：优化 API 调用逻辑

**核心思想**：使用正确的 API 调用顺序

**API 调用关系**：

| 函数 | 功能 | 输入 | 输出 |
|------|------|------|------|
| `search_album()` | 搜索专辑 | `artist`, `album_name` | `albummid` |
| `get_album_info()` | 获取详细信息 | `albummid` | `pub_year`, `pub_date`, `genre` 等 |

**正确的调用顺序**：

```python
# 步骤 1：搜索获取 albummid
album_info = search_album(artist, album_name)
if album_info and "albummid" in album_info:
    albummid = album_info["albummid"]
    
    # 步骤 2：获取详细信息
    album_details = get_album_info(albummid)
    
    if album_details:
        pub_year = album_details["pub_year"]
        # ...
else:
    logger.warning("未找到 albummid")
    logger.warning("跳过专辑：未找到发行年份")
```

**关键改进**：
- ✅ 先用 `search_album()` 获取 `albummid`
- ✅ 再用 `get_album_info(albummid)` 获取详细信息
- ✅ 如果找不到 `albummid`，则跳过年份和封面处理
- ✅ 添加了详细的日志输出
