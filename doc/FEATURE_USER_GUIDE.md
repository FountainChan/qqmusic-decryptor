# 封面和发行年份功能 - 使用指南

## 功能说明

### 自动添加的元数据

解密成功后，FLAC 文件会自动包含以下元数据：

1. **音轨号（TRACKNUMBER）**
   - 从文件名提取：`01 歌曲.flac` → `TRACKNUMBER=1`
   
2. **发行年份（DATE）**
   - 从 QQ Music API 获取：`2023-11-26` → `DATE=2023`
   
3. **专辑封面（PICTURE）**
   - 从 QQ Music API 下载并嵌入到 FLAC 文件
   
4. **封面文件（cover.jpg）**
   - 保存到专辑目录，供音乐播放器显示

### 目录结构示例

```
G:\QQMusic\Decrypted\VipSongsDownload\
└── 周杰伦 - 叶惠美\
    ├── 01 以父之名.flac        # 包含完整元数据
    ├── 02 以父之名.flac        # 共享相同的封面和年份
    ├── 03 以父之名.flac        # 共享相同的封面和年份
    ├── ...
    └── cover.jpg               # 专辑封面文件
```

### FLAC 文件元数据

解密后的 FLAC 文件包含：

```xml
<TAG>
  <ARTIST>周杰伦</ARTIST>
  <ALBUM>叶惠美</ALBUM>
  <ALBUMARTIST>周杰伦</ALBUMARTIST>
  <TITLE>以父之名</TITLE>
  <TRACKNUMBER>1</TRACKNUMBER>      <!-- 音轨号 -->
  <DATE>2003</DATE>                 <!-- 发行年份 -->
</TAG>

<METADATA_BLOCK>
  <PICTURE>
    <MIME_TYPE>image/jpeg</MIME_TYPE>
    <TYPE>3</TYPE>                  <!-- 封面 -->
    <DESCRIPTION>front cover</DESCRIPTION>
    <DATA>...</DATA>              <!-- 封面图片数据 -->
  </PICTURE>
</METADATA_BLOCK>
```

## 工作原理

### 处理流程

```
解密成功
   ↓
1. 提取音轨号（从文件名）
   ├─ 匹配：`01 歌曲.flac` → 1
   └─ 写入：TRACKNUMBER=1
   ↓
2. 读取文件元数据（从 FLAC）
   ├─ 读取：ARTIST = "周杰伦"
   └─ 读取：ALBUM = "叶惠美"
   ↓
3. 查询 QQ Music API（带缓存）
   ├─ 搜索专辑：getSearchByKey
   ├─ 获取信息：getAlbumInfo
   └─ 下载封面：getImageUrl
   ↓
4. 处理专辑信息
   ├─ 提取年份：pub_time → 2003
   ├─ 下载封面：http://i.gtimg.cn/...jpg
   └─ 保存缓存：artist::album → {...}
   ↓
5. 写入文件元数据
   ├─ 添加年份：DATE=2003
   ├─ 嵌入封面：PICTURE 块
   └─ 保存封面：cover.jpg
   ↓
完成
```

### 缓存机制

为节省资源，同一专辑的多首歌会共享信息：

```
同一专辑（周杰伦 - 叶惠美）：
  第 1 首歌 → 查询 API，下载封面（3-6 秒）
  第 2 首歌 → 使用缓存（<1 秒）
  第 3 首歌 → 使用缓存（<1 秒）
  ...
  第 N 首歌 → 使用缓存（<1 秒）

性能提升：节省 80%+ 的 API 调用时间
```

## 常见问题

### Q1：API 连接失败怎么办？

**A**：
1. 检查 API 服务是否运行：`http://192.168.110.194:3200`
2. 检查网络连接
3. 检查防火墙设置
4. 失败时会自动跳过封面和年份，不影响解密

### Q2：封面下载失败会影响解密吗？

**A**：不会。封面下载失败时：
- 音轨号仍然会添加
- 年份仍然会添加（如果获取成功）
- 只是封面和封面文件缺失
- 日志会记录错误信息

### Q3：为什么有些文件没有封面？

**A**：可能的原因：
1. FLAC 文件缺少 ARTIST 或 ALBUM 标签
2. API 搜索不到匹配的专辑
3. 专辑没有封面图片
4. 网络连接问题

### Q4：缓存会持久化吗？

**A**：目前不会。
- 缓存存储在内存中
- 程序重启后缓存清空
- 重新解密时会重新查询 API

### Q5：可以手动添加元数据吗？

**A**：可以。
1. 使用音乐标签编辑器（如：Mp3tag、TagScanner）
2. 手动编辑 FLAC 文件的元数据
3. 解密工具不会覆盖已有的元数据

## 高级功能

### 自定义 API 地址

如果 API 地址不是 `http://192.168.110.194:3200`，可以修改：

**修改位置**：
- `qqmusic_api_client.py` 第 24 行
- 将 `base_url` 改为你的 API 地址

```python
BASE_URL = "http://your-api-address:port"  # 修改这里
```

### 跳过封面和年份

如果只想添加音轨号，不想获取封面和年份：

**方法 1**：修改代码
- 注释掉 `process_album_metadata()` 调用
- 只保留 `add_track_number_to_flac()` 调用

**方法 2**：不启动 API 服务
- API 服务未运行时，会自动跳过封面和年份

## 性能优化建议

### 提高处理速度

1. **使用 Git Bash**：推荐使用 `bash` 而不是 `cmd`
2. **确保网络稳定**：避免封面下载中断
3. **使用 SSD**：加快文件读写速度
4. **关闭杀毒软件**：减少文件扫描时间

### 减少资源占用

1. **解密完成后关闭 GUI**：释放内存
2. **定期清空日志文件**：`logs/decrypt.log`
3. **不使用其他软件**：避免同时运行多个程序

## 故障排除

### 问题：元数据没有添加

**检查清单**：
- [ ] FLA C 文件已解密成功
- [ ] FLAC 文件包含 ARTIST 和 ALBUM 标签
- [ ] API 服务可以访问
- [ ] 网络连接正常
- [ ] 查看日志文件是否有错误

**解决方法**：
1. 检查日志：`logs/decrypt.log`
2. 手动测试 API：访问 `http://192.168.110.194:3200/getSearchByKey?key=周杰伦 叶惠美&remoteplace=album`
3. 重新解密：使用 `--force` 参数（如果有）
4. 手动添加元数据：使用音乐标签编辑器

### 问题：封面文件没有生成

**可能原因**：
- API 没有返回封面 URL
- 封面下载失败
- 目录权限不足

**解决方法**：
1. 检查日志中的错误信息
2. 尝试手动下载封面
3. 检查磁盘空间
4. 检查目录写入权限

### 问题：年份不正确

**可能原因**：
- API 返回的年份格式不正确
- 专辑信息匹配错误
- 使用了错误的专辑版本

**解决方法**：
1. 检查日志中的专辑信息
2. 确认搜索的专辑名正确
3. 手动编辑 FLAC 文件的 DATE 字段
4. 使用音乐标签编辑器修改
