## 🔍 根本原因分析

### 问题根源

**代码逻辑错误**（`supplement_album_metadata.py` 第 533 行）：

```python
# 修改前（错误）
if cover_data:
    save_cover_to_directory(album_dir, cover_data)
```

**参数分析**：
- `album_dir`：歌手目录（`G:/QQMusic/Decrypted/VipSongsDownload/林峯`）
- `audio_files[0]`：第一个音频文件（`林峯\A Time 4 You 新曲+精选\12 爱不疚.flac`）

**问题**：
- `save_cover_to_directory(album_dir, cover_data)` 直接保存到 `album_dir`
- 没有考虑音频文件所在的目录（专辑子目录）
- 导致封面保存到了歌手目录，而不是专辑子目录

---

## 🔧 修复方案

### 方案 A：保存到音频文件所在目录（推荐）

#### 实施步骤

1. **修改封面保存逻辑**（`supplement_album_metadata.py` 第 533-536 行）

**修改前**：
```python
    # 保存封面到目录
    if cover_data:
        save_cover_to_directory(album_dir, cover_data)
    
    return stats
```

**修改后**：
```python
    # 保存封面到音频文件所在的目录（专辑子目录）
    if cover_data:
        # 获取第一个音频文件所在的目录（专辑子目录）
        first_audio_dir = os.path.dirname(audio_files[0])
        
        # 保存封面
        cover_path = save_cover_to_directory(first_audio_dir, cover_data)
        
        # 添加详细日志
        logger.info(f"封面已保存到: {cover_path}")
    
    return stats
```

**优点**：
- ✅ **通用性好**：适用于所有目录结构
  - 如果专辑在子目录中（`歌手\专辑\`），封面保存到专辑目录
  - 如果专辑直接在歌手目录中（`歌手\`），封面保存到歌手目录
- ✅ **无需配置**：用户不需要手动指定目录
- ✅ **符合预期**：对于 `歌手\专辑\` 的结构，封面会保存到专辑目录

**缺点**：
- ⚠️ **封面总是保存到第一个音频文件所在的目录**
  - 如果专辑有多个子目录，封面只会保存到第一个子目录
