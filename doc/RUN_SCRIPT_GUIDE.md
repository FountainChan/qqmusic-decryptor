# 一键运行脚本使用指南

## 🚀 快速开始

### 方法 1：双击运行（推荐）

1. **修改默认目录**（可选）
   - 打开文件：`run_supplement.bat`
   - 找到第 17 行：`set DEFAULT_INPUT_DIR=G:\QQMusic\Decrypted`
   - 修改为你的解密目录路径

2. **双击运行**
   - 找到文件：`run_supplement.bat`
   - 双击运行
   - 等待处理完成

### 方法 2：拖动目录到脚本上

1. **打开文件管理器**
   - 找到文件：`run_supplement.bat`
   
2. **拖动专辑目录**
   - 将专辑目录拖动到 `run_supplement.bat` 文件上
   - 自动处理该目录下的所有 FLAC 文件

3. **释放鼠标**
   - 脚本自动开始处理

### 方法 3：修改后双击

1. **编辑脚本**
   - 右键 `run_supplement.bat` → "编辑"
   
2. **修改默认目录**
   - 找到第 17 行
   - 修改为你想要处理的目录路径
   
3. **保存并运行**
   - 保存文件
   - 双击运行

## 📝 脚本功能

### 自动检测

✅ **Python 安装检查**
- 自动检测 Python 是否安装
- 如果未找到，显示下载链接

✅ **目录验证**
- 检查目录是否存在
- 检查是否是目录（不是文件）
- 如果无效，显示错误信息

✅ **路径处理**
- 自动检测是拖动还是默认路径
- 支持相对路径和绝对路径

### 处理功能

✅ **单专辑处理**
- 自动扫描专辑目录中的所有 FLAC 文件
- 获取专辑信息（ARTIST、ALBUM）
- 调用 API 获取封面和年份
- 嵌入元数据到所有 FLAC 文件
- 保存封面为 cover.jpg

✅ **批量处理（根目录）**
- 自动遍历根目录下的所有子目录
- 每个子目录作为一个专辑处理
- 智能缓存避免重复 API 请求
- 显示每个专辑的处理进度

✅ **智能缓存**
- 基于 ARTIST::ALBUM 的缓存键
- 同一专辑多首歌只查询一次 API
- 内存缓存，重启后清空

## ⚙️ 自定义配置

### 修改默认目录

1. 右键 `run_supplement.bat` → "编辑"
2. 找到第 17 行
3. 修改为你的目录路径：
   ```batch
   set DEFAULT_INPUT_DIR=G:\QQMusic\Decrypted
   ```
4. 保存文件

### 支持的目录路径

- ✅ **绝对路径**：
  - Windows 路径：`G:\QQMusic\Decrypted`
  - UNC 路径：`\\Server\Share\Music`

- ✅ **相对路径**：
  - 相对于脚本目录：`.\Decrypted`
  - 相对于父目录：`..\Decrypted`

- ✅ **带空格的路径**：
  - 自动处理：`G:\My Music\Decrypted`

## ⚠️ 注意事项

### Python 环境

**要求**：
- Python 3.6+
- requests 库：`pip install requests`
- mutagen 库：`pip install mutagen`

**检查**：
- 打开命令提示符（CMD）
- 输入：`python --version`
- 应该显示：`Python 3.x.x`

### API 服务

**要求**：
- API 地址：`http://192.168.110.194:3200`
- 网络连接正常
- 防火墙允许访问

**测试**：
- 在浏览器中访问：`http://192.168.110.194:3200/getAlbumInfo?albummid=000ILKG63fED7K`
- 应该返回 JSON 数据

### FLAC 文件

**要求**：
- 文件格式：`.flac`
- 包含标签：`ARTIST`、`ALBUM`
- 文件可读写

**检查**：
- 使用音频播放器查看 FLAC 文件的元数据
- 应该能看到 ARTIST 和 ALBUM 标签

## 🔧 故障排除

### 问题 1：双击后立即退出

**可能原因**：
- 默认目录不存在
- 默认目录是文件而不是目录

**解决方法**：
1. 编辑 `run_supplement.bat`
2. 修改默认目录为存在的目录
3. 保存并重新运行

### 问题 2：找不到 Python

**错误信息**：
```
错误：未找到 Python

请确保 Python 已安装并添加到 PATH 环境变量中

下载地址：https://www.python.org/downloads/
```

**解决方法**：
1. 下载并安装 Python 3.6+
2. 添加 Python 到 PATH 环境变量
3. 重启命令提示符
4. 重新运行脚本

### 问题 3：处理失败

**错误信息**：
```
处理失败，错误代码：1
```

**可能原因**：
- API 连接失败
- 网络连接问题
- FLAC 文件损坏

**解决方法**：
1. 检查网络连接
2. 检查 API 服务是否运行
3. 检查 FLAC 文件是否损坏

### 问题 4：封面和年份没有添加

**可能原因**：
- FLAC 文件缺少 ARTIST 或 ALBUM 标签
- API 搜索不到专辑

**解决方法**：
1. 使用音频播放器检查 FLAC 文件的元数据
2. 检查 ARTIST 和 ALBUM 标签是否存在
3. 尝试手动编辑 FLAC 文件添加标签

### 问题 5：部分文件处理失败

**原因**：
- 文件被占用
- 文件权限问题

**解决方法**：
1. 关闭所有音频播放器和编辑器
2. 以管理员权限运行脚本
3. 检查文件权限

## 🎉 使用技巧

### 技巧 1：分批处理

**场景**：专辑数量很多（100+）

**操作**：
1. 将专辑分到多个目录
2. 为每个目录创建一个 `.bat` 文件
3. 或者多次编辑默认目录并运行

**优势**：
- 避免一次性处理时间过长
- 可以分阶段查看结果
- 失败时容易定位问题

### 技巧 2：先测试

**场景**：首次使用，不确定是否正常

**操作**：
1. 先处理一个专辑（拖动）
2. 检查结果是否正确
3. 检查 FLAC 文件的元数据
4. 确认后批量处理所有

**优势**：
- 快速验证功能
- 发现问题及时解决
- 避免批量处理浪费时间

### 技巧 3：保留日志

**场景**：需要记录处理日志

**操作**：
1. 右键 `run_supplement.bat` → "编辑"
2. 在脚本最后添加重定向命令：
   ```batch
   python "%SCRIPT_DIR%\supplement_album_metadata.py" "%INPUT_DIR%" >> log.txt 2>&1
   ```
3. 保存并运行

**优势**：
- 所有输出都保存到文件
- 方便查看历史记录
- 方便调试问题
